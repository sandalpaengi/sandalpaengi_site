# 역할
너는 세계적인 수준의 웹 개발자이자 UI/UX 디자이너야. 
나는 1인 목공방을 운영하는 초보자이고, 코딩을 전혀 몰라.

# 프로젝트 목표
나를 위해 'Next.js (App Router)'를 사용하여 목공방 웹사이트를 만들어줘.
최종적으로 Vercel에 배포할 예정이야.
1인 목공방의 '선제작 후판매(Drop)' 방식 웹사이트 구축.

## 핵심 철학 및 운영 방식
- **운영자**: 1인 목공방 작가 (개발 지식 없음, 수동 관리 선호)
- **판매 방식**: 선제작 후판매 (Drop). 작품 하나하나가 희소성이 있음.
- **기술적 목표**: 
  - 데이터베이스(DB)나 복잡한 관리자 페이지 절대 금지.
  - 모든 데이터는 `data/products.json` 파일 하나로 사장님이 직접 수정 관리한다.
  - 결제 시스템 자동 연동 안 함 (결제 링크만 연결).

# 사이트 요구사항
1. **디자인 컨셉**: 
   - 목공방의 따뜻함과 신뢰감이 느껴지는 '미니멀리즘' 디자인.
   - 모바일에서도 보기 편한 '반응형 웹'이어야 함.
- **핵심 키워드**: Organic Minimalism (자연스러운 미니멀리즘), Storytelling, Craftsmanship.
- **분위기**: 
  - 너무 딱딱하거나 차가운 기업 느낌(Corporate look)을 지양할 것.
- **모션/애니메이션**:
  - 화려한 스크롤 효과는 배제할 것. (성능 최적화 및 심플함 추구)
  - 대신 버튼에 마우스를 올렸을 때 색이 부드럽게 변하거나, 이미지가 살짝 확대되는 정도의 **'마이크로 인터랙션(Micro-interactions)'**만 최소한으로 적용.


2. **필수 페이지 구성**:
   - **메인(Home)**: 감성적인 대표 사진이 크게 들어가고, 브랜드 소개 문구 배치.
   - **포트폴리오(Gallery)**: 내가 만든 가구 사진들을 격자(Grid) 형태로 보여줌. 클릭하면 상세 사진이 뜸.
   - **블로그(Blog)**: 작업 일지나 가구 관리법 등을 올릴 게시판 (MDX 파일 기반으로 쉽게 작성 가능하게).
   - **상점(Shop)**: 상품 리스트가 보이는 페이지 (일단 결제 기능 없이 상품명/가격/문의하기 버튼만 배치).
   - **문의(Contact)**: 내 이메일, 전화번호, 공방 위치(지도), 인스타그램 링크 표시.

3. **데이터 관리 및 기능 요구사항 (중요!)
- **데이터베이스 없음**: 별도 DB 없이 `data/products.json` 파일 하나로 상품 데이터를 관리한다.
- **재고 관리 로직**:
  - 나는 '세상에 하나뿐인 가구'를 판다.
  - JSON 데이터에 `stock` (재고 수량) 필드를 만든다.
  - `stock: 0`이면 상품 상세 페이지의 구매 버튼이 비활성화되고, 'Sold Out(작품 소장됨)'이라는 문구가 떠야 한다.
  - 상품 목록 페이지에서도 품절된 상품은 이미지 채도를 낮추거나 'Sold Out' 배지를 달아준다.




3. **기술 스택 (이대로 짜줘)**:
   - 프레임워크: Next.js 14 이상 (App Router 사용)
   - 스타일링: Tailwind CSS (디자인 수정이 쉽게)
   - 아이콘: Lucide React
   - 이미지: next/image 최적화 사용
   - 데이터: 로컬 JSON 파일 기반

# 실행 지시
프로젝트 폴더 구조를 먼저 잡아주고, 그 다음 필요한 패키지 설치 명령어와 각 파일의 코드를 순서대로 작성해줘. 초보자인 내가 따라 할 수 있도록 단계별로 설명해줘.

# 디자인 전면 수정 요청 (Re-design)

방금 만든 디자인도 좋지만, 내가 원하는 방향으로 스타일을 완전히 바꾸고 싶어.
레퍼런스 사이트(reidschwartz.net)의 'Raw & Industrial' 감성을 벤치마킹하여 다시 잡아줘.

## 1. 핵심 컨셉 변경
- **기존**: 부드럽고 따뜻한 Organic Minimalism
- **변경**: **"Utilitarian Brutalism" (실용적 브루탈리즘)** & **"Classic Editorial"**
  - 장식적인 요소를 모두 제거하고, **'흰 배경 + 검은 글씨 + 날것의 사진'**만 남긴다.
  - 여백을 극단적으로 많이 사용하지 말고, 정보 전달에 충실한 단단한 레이아웃을 사용한다.

## 2. 메인 페이지 레이아웃 (가장 중요)
- **Hero Image 제거**: 화면을 가득 채우는 큰 배너 이미지를 없앤다.

- **이미지 그리드**:
  - 텍스트 아래에 바로 작품 사진들이 **2열 또는 3열 그리드**로 투박하게 나열된다.
  - 사진 간격(Gap)은 좁게, 모서리는 둥글지 않고 **직각(0px radius)**으로 처리한다.

## 3. 타이포그래피 & 컬러
- **폰트**:
  - 시스템 기본 **Serif (명조 계열)** 또는 **Mono (타자기체)**를 사용하여 '문서' 같은 느낌을 준다.
  - 구글 폰트의 'EB Garamond' (우아함) 또는 'Courier Prime' (투박함) 추천.
- **컬러**:
  - 배경: 완전한 흰색 (#FFFFFF) 또는 아주 연한 회색 (#F5F5F5).
  - 텍스트: 완전한 검정 (#000000).
  - 포인트 컬러: **없음**. (링크에 밑줄 정도만 사용)

## 4. 네비게이션 (Header)
- 로고는 왼쪽 상단, 메뉴(Shop, About, Contact)는 오른쪽 상단에 텍스트로만 심플하게 배치.
- 헤더 배경색을 투명하게 하거나 흰색으로 고정.

위 가이드를 바탕으로 **`layout.tsx` (헤더/폰트)와 `page.tsx` (메인 화면) 코드를 전면 수정해줘.**

# 폰트 변경 요청

현재 적용된 'EB Garamond' 폰트가 조금 고전적인 것 같아.
사이트 분위기를 더 모던하고 중립적인 'Swiss Style'로 가고 싶어.

1. **메인 폰트 변경**:
   - 구글 폰트의 **'Inter'**로 변경해줘. (font-sans에 적용)
   - 헬베티카(Helvetica)와 가장 비슷한 느낌으로 굵기(font-weight)를 조절해줘.

2. **포인트 폰트 유지**:
   - 'Courier Prime' (타자기체)은 기술적인 스펙이나 날짜 등을 표시할 때 포인트로 계속 쓰고 싶으니 남겨둬.

3. **스타일 조정**:
   - 제목(Headings)은 아주 굵게(Bold) 또는 아주 얇게(Light) 처리해서 대비를 줘.
   - 본문 텍스트 간격(Letter-spacing)을 살짝 좁혀서 단단한 느낌을 줘 (tracking-tight).

   # 디자인 및 레이아웃 수정 요청

현재 디자인에서 폰트를 교체하고, 메인 화면 구성을 변경하고 싶어.

## 1. 폰트 재설정 (Mix & Match)
- **Sans-serif (기본 폰트)**: 현재의 **'Inter'**를 그대로 유지해줘.
- **Monospace (포인트 폰트)**: 기존 'Courier Prime'을 삭제하고, **'IBM Plex Mono'** (Google Fonts)로 교체해줘.
- **적용 규칙 변경**:
  - **상품 제목 & 가격**: 기존 설정과 달리, 가독성을 위해 **'Inter' (font-sans)**를 적용해줘. (타자기체 쓰지 말 것)
  - **스펙/날짜/코드**: 이런 기술적인 정보에만 **'IBM Plex Mono' (font-mono)**를 사용해줘.

## 2. 홈 화면 (Page.tsx) 개편
- **상품 목록 제거**: 메인 화면(`app/page.tsx`)에서는 판매 상품(Works) 리스트를 **아예 숨겨줘.**
- **블로그 중심 배치**:
  - 메인에는 '작가의 인트로(Manifesto)'와 **'최신 블로그 글(Journal)'** 리스트만 보이게 해줘.
  - 방문자가 "물건 파는 곳"이 아니라 "이야기가 있는 공방"에 왔다는 느낌을 받도록.

## 3. 헤더 (Header) 기능 수정
- **로고 링크**: 왼쪽 상단 로고(텍스트)를 클릭하면 무조건 **메인 페이지(`/`)**로 이동하도록 `<Link>` 컴포넌트를 연결해줘.

위 내용을 반영해서 `layout.tsx` (폰트), `page.tsx` (메인화면), `components/Header.tsx` (로고), `components/ProductCard.tsx` (폰트 적용) 파일을 수정해줘.
